<h4 class="upper ral fat gray">Options</h4><br>
<dl class="options">
	<dt class="Ctext">pageTitle</dt>
	<dd>Text which gives a good description of the page. Longer than a display head, shorter than a summary. Think of it as a multimedia headline. This gets used to build a standard tweet for the share buttons.</dd>
	
	<dt class="Ctext">folioTitle</dt>
	<dd>Text which gives short description of the page. Preferably a display head. This gets used on the floating folio.</dd>
	
	<dt class="Ctext">shortUrl</dt>
	<dd>A short URL that will be used in the pregenerated tweet.</dd>
	
	<dt class="boolean">flat</dt>
	<dd>A boolean that activates the static folio that appears on top of the page.</dd>
	
	<dt class="boolean">floater</dt>
	<dd>A boolean that activates the floating folio that appears as you scroll the page.</dd>
	
	<dt class="boolean">tracker</dt>
	<dd>A boolean that activates the progress bar on the floating folio that updates as you scroll the page.</dd>

<p>Story Content can be hard coded into a story via vanilla html, or you can call it in via ajax from a live story with a static html fallback. The next few paragraphs were called in this way:</p>

<p>Here's how we called in the above info:</p>
<pre class="code"><span class="comment">// A DEFERRED OBJECT</span>
$load = new $.Deferred();
<span class="comment">LOAD THE CONTENT</span>
$('.test-box').loadContent({
	<span class="Ctext">'heraldUrl'</span> : <span class="Ctext">'http://www.miamiherald.com/2013/12/30/3844822/miami-dade-leaders-asks-new-years.html'</span>,
	<span class="Ctext">'fallback'</span> : <span class="Ctext">'data/data.html'</span>,
	<span class="Ctext">'childExpected'</span> : <span class="Ctext">'p'</span>
});
<span class="comment">RESOLVE THE OBJECT</span>
$load.resolve();
<span class="comment">// BEGIN LAYOUT CODE BLOCK</span>
$.when($load1).then( function () {
	console.log('done')
	<span class="comment">// RUN LAYOUT LOGIC HERE</span>
})
</pre>
<p>We began the load process by creating a <a href="http://api.jquery.com/category/deferred-object/" target="blank">deferred object</a> that gets resolved after we load our content. That way, we can ensure that any layout logic we write happens after the content has finished loading. Then, we load the content via the .loadContent method, and we resolve the deferred object right after. In the above example, we tried loading this Herald <a href="http://www.miamiherald.com/2013/12/30/3844822/miami-dade-leaders-asks-new-years.html" target="blank">story</a>, but the scrape failed, so it defaulted to the static HTML <a href="http://www.miamiherald.com/static/media/projects/libraries/sinclair/data/data.html" target="_blank">fallback</a> that you specified. </p>
<h4 class="upper ral fat gray">Options</h4><br>
<dl class="options">
	<dt class="Ctext">heraldUrl</dt>
	<dd>The full-version Url of an online story to scrape data from. It must be the non-paginated version of the story.</dd>
	<dt class="Ctext">fallback</dt>
	<dd>A static HTML fallback file to load in case of an AJAX error.</dd>
	<dt class="Ctext">childExpected</dt>
	<dd>The type of child element you are loading. Most likely paragraphs.</dd>
	
</dl>

<h3 class="ral upper fat large">Preparing for Layout</h3>
<p>Story elements can be added to a story in a variety of ways, since most story building utilities are invoked on a DOM element. When dealing with a story, it's usually best to define a variable for all paragraphs in a story, and then build the layout logic based on paragraphs.</p>
<pre class="code">$story = $('#article-container p')
</pre>
<p>Now, we can target different story paragraphs using jQuery's <a href="http://api.jquery.com/eq/" target="_blank">.eq()</a> method:</p>

<pre class="code"><span class="comment">// DEFINE THE STORY PARAGRAPH VARIABLE</span>
$story = $('#test-2 p')
<span class="comment">// MAKE FIRST GRAF RED</span>
$story.eq(0).css('color','#f00');
<span class="comment">// MAKE SECOND GRAF GREEN</span>
$story.eq(1).css('color','#0f0');
</pre>
